La blockchain post-quantique.
Simple. Rapide. Déflationniste.

ChainGO protège chaque transaction avec ML-DSA-65, le standard de signature résistant aux ordinateurs quantiques (NIST FIPS 204). Frais brûlés automatiquement, tokens et contrats créés sans une ligne de code, et un réseau pair-à-pair que n'importe qui peut rejoindre.

Les piliers

Le projet que nous portons

Les blockchains actuelles signent avec une cryptographie (ECDSA, Ed25519) que les ordinateurs quantiques sauront casser. ChainGO est conçue dès le premier bloc pour ce monde-là — sans sacrifier la vitesse ni la simplicité.

Sécurité post-quantique Actif

Chaque wallet, transaction et bloc est signé en ML-DSA-65 (FIPS 204, niveau NIST 3). Hachage SHA3-256. Aucune courbe elliptique vulnérable au quantique, nulle part.

Vitesse Actif

31 000 transactions/seconde mesurées (vérification + exécution) grâce à la vérification parallèle sur tous les cœurs. Blocs toutes les 500 ms, confirmation en ~1 seconde. Option tip pour passer en priorité.

Économie déflationniste Actif

Les frais de base sont détruits (burn) à chaque transaction, ajustés automatiquement à la congestion (modèle EIP-1559). Plus le réseau est utilisé, plus le CGO se raréfie.

Tokens & contrats no-code Actif

Créez votre token et déployez des smart contracts sans code depuis le studio : vesting, escrow, multisig M-of-N et DAO. Pour les devs, une VM WASM exécute des contrats arbitraires (Rust, AssemblyScript…) — câblée sur le testnet.

Anonymat fort R&D

Pseudonyme aujourd'hui ; un circuit de transaction blindée zk-STARK post-quantique est déjà fonctionnel en R&D (montants et destinataire cachés, sécurité hash-only). Prototype maison, non audité, hors-consensus — audit communautaire ouvert.

Décentralisation Actif

Réseau pair-à-pair : chaque nœud se synchronise et relaie. Validez avec 10 000 CGO ou déléguez dès 1 CGO — le tirage des blocs est pondéré par le stake, et des rounds de secours garantissent que la chaîne avance même si des validateurs tombent.

Le fonctionnement

Comment ça marche

Le cycle de vie d'une transaction, sans jargon.

1

Un wallet = une paire de clés post-quantiques

Votre wallet génère une clé ML-DSA-65. Votre adresse (cg…) est l'empreinte SHA3 de la clé publique. La clé privée reste chez vous, chiffrée (scrypt + AES-256-GCM).

2

Vous signez votre transaction

Transfert, création de token, stake, délégation… Vous fixez votre tip (pourboire au validateur — enchère libre pour la priorité) et votre plafond de frais. La signature prouve que c'est vous, et personne — même avec un ordinateur quantique — ne peut la falsifier.

3

Le réseau la propage et la vérifie

Votre transaction entre dans la file d'attente (mempool) et se diffuse de nœud en nœud. Chaque nœud revérifie la signature en parallèle sur tous ses cœurs — c'est ce qui permet les dizaines de milliers de TPS.

4

Un validateur la grave dans un bloc

Toutes les 500 ms, le consensus Aurora tire au sort un validateur (proportionnellement à son stake + délégations) qui assemble le bloc — meilleurs tips d'abord. S'il est hors-ligne, un validateur de secours prend le relais automatiquement.

5

Les frais brûlent, validateur et délégateurs sont récompensés

Le frais de base de chaque transaction est détruit pour toujours (déflation). Le validateur touche les tips + une émission d'environ 3 %/an, partagée au pro-rata avec ses délégateurs. Votre transaction est finale : solde mis à jour sur tous les nœuds du monde.

Feuille de route

Où en est ChainGO

Transparente — y compris sur ce qui n'existe pas encore. Le testnet est en ligne ; le mainnet ouvre après l'audit externe et des validateurs indépendants.

Phase 1 — Fondations Livré

Chaîne post-quantique complète : consensus PoS Aurora avec rounds de secours, frais EIP-1559 brûlés, staking + délégation + unbonding, tokens no-code, premiers smart contracts, wallets, P2P, API REST, 31 000 TPS au bench.

Phase 2 — Consensus BFT & slashing En cours · ~94 %

Finalité multi-validateurs (2f+1), set de validateurs figé par hauteur, verrouillage POL, slashing complet (double-signature + inactivité), fork-choice + reorg testé en partition, codec binaire, fuzzing. Reste : audit externe et des validateurs indépendants.

Phase 4 — Smart contracts Livré · testnet

Vesting, escrow, multisig et DAO déployables depuis le studio. Moteur WASM (contrats arbitraires) câblé en consensus sur le testnet : on déploie du bytecode et on l'appelle depuis le studio, la CLI ou l'API. Déterminisme garanti (gas instrumenté, opcodes restreints, interpréteur). Activé sur mainnet après audit.

Phase 5 — Écosystème & outils En cours

En ligne : explorateur, studio no-code, dashboard validateur, banc d'essai, wallet web. Les SDK JavaScript & Python ont leurs repos dédiés (signature ML-DSA-65 identique au nœud) ; packages npm/PyPI à venir. La suite : documentation complète et présence communautaire.

Phase 3 — Anonymat fort R&D · prototype

Pile zk-STARK maison post-quantique + un circuit de transaction blindée fonctionnel : prouve en zero-knowledge une dépense valide, montants et destinataire cachés, sécurité hash-only (zéro courbe). Fait-maison, non audité, hors-consensus — audit communautaire ouvert.

Démarrer

Démarrer en 60 secondes

Un binaire, zéro dépendance — Windows, Linux, macOS (et Docker pour les serveurs).

# Lancer un nœud devnet (validateur + faucet inclus)
chaingo node start --dev

# Créer un wallet post-quantique
chaingo wallet new alice

# Recevoir des CGO de test, puis transférer
chaingo faucet --to alice --amount 500
chaingo send --from alice --to cg… --amount 42.5 --fast

# Créer SON token, sans code — ou déléguer dès 1 CGO
chaingo token create --from alice --symbol MONTOK --name "Mon Token" --supply 1000000
chaingo delegate --from alice --to cg… --amount 50

Pas envie de ligne de commande ? Créez un wallet dans le navigateur ou parcourez la chaîne en direct. Pour faire tourner un nœud ou intégrer ChainGO : Référence API · code source.

Le testnet public est en ligne — explorez-le librement, obtenez des CGO de test via le faucet, rejoignez le réseau. Le mainnet ouvrira après l'audit de sécurité externe et la finalisation de la Phase 2 (consensus BFT durci). L'anonymat fort (zk-STARK) arrive en Phase 3.

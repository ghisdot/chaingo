Le projet que nous portons

Les blockchains actuelles signent avec une cryptographie (ECDSA, Ed25519) que les ordinateurs quantiques sauront casser. ChainGO est conçue dès le premier bloc pour ce monde-là — sans sacrifier la vitesse ni la simplicité.

🛡️ Sécurité post-quantique ACTIF

Chaque wallet, transaction et bloc est signé en ML-DSA-65 (FIPS 204, niveau NIST 3). Hachage SHA3-256. Aucune courbe elliptique vulnérable au quantique, nulle part.

⚡ Vitesse ACTIF

31 000 transactions/seconde mesurées (vérification + exécution) grâce à la vérification parallèle sur tous les cœurs. Blocs toutes les 500 ms, confirmation en ~1 seconde. Option tip pour passer en priorité.

🔥 Économie déflationniste ACTIF

Les frais de base sont détruits (burn) à chaque transaction, ajustés automatiquement à la congestion (modèle EIP-1559). Plus le réseau est utilisé, plus le CGO se raréfie.

🪙 Tokenisation no-code ACTIF

N'importe qui crée son token en une commande ou un appel API — nom, supply, décimales, mintable ou non. 10 CGO brûlés, zéro ligne de code, zéro audit de contrat à payer.

🕶️ Anonymat renforcé PHASE 3

Aujourd'hui : pseudonymat (adresses non nominatives, historique public). Demain : transferts confidentiels par preuves zk-STARK — la seule famille de preuves à divulgation nulle résistante au quantique — et adresses furtives.

🌐 Décentralisation ACTIF

Réseau pair-à-pair : chaque nœud se synchronise et relaie. Devenez validateur en stakant 10 000 CGO — le tirage des blocs est pondéré par le stake, et des rounds de secours garantissent que la chaîne avance même si des validateurs tombent.

Comment ça marche

Le cycle de vie d'une transaction, sans jargon.

Un wallet = une paire de clés post-quantiques

Votre wallet génère une clé ML-DSA-65. Votre adresse (cg…) est l'empreinte SHA3 de la clé publique. La clé privée reste chez vous, chiffrée (scrypt + AES-256-GCM).

Vous signez votre transaction

Transfert, création de token, stake… Vous fixez votre tip (pourboire au validateur — enchère libre pour la priorité) et votre plafond de frais. La signature prouve que c'est vous, et personne — même avec un ordinateur quantique — ne peut la falsifier.

Le réseau la propage et la vérifie

Votre transaction entre dans la file d'attente (mempool) et se diffuse de nœud en nœud. Chaque nœud revérifie la signature en parallèle sur tous ses cœurs — c'est ce qui permet les dizaines de milliers de TPS.

Un validateur la grave dans un bloc

Toutes les 500 ms, le consensus Aurora tire au sort un validateur (proportionnellement à son stake) qui assemble le bloc — meilleurs tips d'abord. S'il est hors-ligne, un validateur de secours prend le relais automatiquement.

Les frais brûlent, le validateur est récompensé

Le frais de base de chaque transaction est détruit pour toujours (déflation). Le validateur touche les tips + une émission d'environ 3 %/an calculée sur le stake total. Votre transaction est finale : solde mis à jour sur tous les nœuds du monde.

Feuille de route

Transparente, y compris sur ce qui n'existe pas encore.

Phase 1 — Fondations LIVRÉ

Chaîne post-quantique complète : consensus PoS Aurora avec rounds de secours, frais EIP-1559 brûlés, staking + unbonding, tokens no-code, wallets, P2P, API REST, 31 000 TPS au bench.

Phase 2 — Consensus BFT & slashing

Votes de finalité multi-validateurs (2f+1), pénalités pour validateurs malhonnêtes, arbre de Merkle creux, codec binaire compact.

Phase 3 — Anonymat fort

Transferts confidentiels zk-STARK (résistants au quantique) + adresses furtives. Le mode private actuel deviendra un vrai bouclier de confidentialité.

Phase 4 — Smart contracts no-code

Templates déclaratifs (vesting, escrow, multisig, DAO) déployables en un clic, puis VM WASM déterministe pour les développeurs.

Phase 5 — Explorateur & SDK

Explorateur de blocs public, SDK JavaScript et Python, documentation interactive.

Démarrer en 60 secondes

Un binaire, zéro dépendance.

# Lancer un nœud devnet (validateur + faucet inclus)
chaingo node start --dev

# Créer un wallet post-quantique
chaingo wallet new alice

# Recevoir des CGO de test, puis transférer
chaingo faucet --to alice --amount 500
chaingo send --from alice --to cg… --amount 42.5 --fast

# Créer SON token, sans code
chaingo token create --from alice --symbol MONTOK --name "Mon Token" --supply 1000000
⚠️ ChainGO est en phase devnet : logiciel expérimental, réseau de test. N'y placez aucune valeur réelle. Le mode « private » est une voie réservée — l'anonymat fort (zk-STARK) arrive en Phase 3.
